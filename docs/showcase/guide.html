Positioning

产品定义

一句话

我们把 OKX Skills 提供的底层链上能力，封装成 AI 可以直接购买的决策工作流。 对调用方 AI 来说，拿到的不是零散技能，而是 policy / trade / exit / watch / machineView。

核心能力

  • Free Opportunity Scan：免费 shortlist
  • Premium Trade Plan：带 Treasury Policy 的入场方案
  • Premium BYO Thesis Plan：验证外部 AI thesis
  • Premium Exit Plan：分批退出与后续监控

本地体验

npm test
npm run preflight
npm run premium:demo
npm run premium:live-demo-thesis

English

English Overview

Positioning

We package OKX Skills into guarded AI decision workflows, so buyers consume a complete result instead of stitching together raw onchain primitives by themselves.

What Buyers Get

  • Free shortlist before payment
  • Policy-gated entry plans
  • Thesis validation for external AI
  • Exit and watch plans after unlock

Runtime Shortcuts

npm run premium:live-smoke
npm run premium:real-payment-thesis-smoke
npm run premium:live-demo-thesis
npm run premium:live-demo-trade

Skill Layer

工作流能力层

高阶意图

  • find_opportunities
  • check_risk_and_prepare_trade
  • prepare_exit_plan
  • track_execution_status
  • buy_premium_research

设计原则

  • 先组合 signal / token / holders / quote，再做建议
  • 先返回 Treasury Policy，再决定是否执行
  • 所有 premium 结果都要先走 402 / verify / settle
  • 同时输出人类说明和 machineView JSON

为什么值钱

我们不是卖底层步骤，而是卖决策结果本身。调用方 AI 不用自己补 policy、entry、exit 和 watch。

Integration

接口与接入

发现与支持

  • GET /supported：支付网络和资产能力
  • GET /manifest：能力、定价和接入元数据

免费入口

  • GET /opportunities：免费 shortlist，不要求支付

Premium 工作流

  • GET /premium/opportunities
  • POST /premium/thesis-plan
  • POST /premium/trade-plan

这些接口都会先返回 402 Payment Required，支付完成后再返回结果。

接入方式

  • 通过 /manifest 做服务发现
  • 通过 openapi.yaml 接入标准接口
  • 通过 SKILL.md 复用高阶意图

Value

价值主张

核心结论

我们把 OKX Skills 组织成 AI 愿意付费调用的决策工作流。 调用方 AI 不必自己拼技能，就能直接拿到带风控的链上决策与执行结果。

为什么成立

  • 调用方 AI 不会天然拥有稳定的 policy / exit / watch 逻辑
  • 它们也不想自己处理 x402 支付、解锁和 machineView 结构
  • 所以直接购买“结果”，比自己拼“步骤”更省成本

商业模式边界

  • 普通 BYO Thesis：直接购买我们的验证与执行服务
  • 白名单 Provider Thesis：平台 20% / Provider 80%
  • 当前分账模式：offchain-ledger
